# ISQ Agent Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a two-workflow n8n system that accepts a blank ISQ PDF via webhook, uses an AI Agent with RAG tools to answer each question from Northstar Labs' knowledge base, and returns a structured JSON response with per-answer confidence scores and review flags.

**Architecture:** Workflow 1 (Knowledge Ingestion) chunks and embeds all 9 knowledge documents into two named in-memory vector stores (`policies-store` and `isqs-store`) at startup. Workflow 2 (ISQ Processing) receives an ISQ PDF via webhook, extracts questions using an LLM, then runs an AI Agent per question — the agent calls `search_policies` and `search_previous_isqs` tools before generating each answer. LLM provider (Ollama or Anthropic) is controlled by a single Config node at the top of Workflow 2, with a Switch node routing to parallel AI Agent nodes per question.

**Tech Stack:** n8n (Docker, localhost:5678), Ollama (llama3.2 + nomic-embed-text, localhost:11434), n8n Simple Vector Store (in-memory), Anthropic Claude (optional, switchable via Config node)

- [ ] **Step 4: Add Basic LLM Chain to extract questions**

Under **AI → Chains**, add **"Basic LLM Chain"**.

Configure the prompt (click the prompt field, switch to expression mode):

```
You are processing an Information Security Questionnaire (ISQ).

Extract every question from the document text below. Return ONLY a valid JSON array of strings. Each string is one complete question as it appears in the document. Do not include section headers, field labels, or instructions — only the questions.

Example format:
["Do you maintain a formal information security policy?", "Is MFA enforced for all staff?"]

Return only the JSON array. No explanation, no markdown.

Document text:
{{ $json.text }}
```

Connect sub-node:
- **AI → Language Models → "Ollama Chat Model"**: Credential `Ollama Local`, Model `llama3.2`

> We use Ollama for question extraction regardless of the Config node — this is a simple parsing task, not the main answer generation. Only the AI Agent (Task 5) is switchable.

- [ ] **Step 5: Add Code node to parse questions into items**

Add a **Code** node after the Basic LLM Chain. Mode: **"Run Once for All Items"**.

```javascript
const rawOutput = $input.first().json.output || $input.first().json.text || '';

// Strip markdown code fences if the model wrapped the output
const cleaned = rawOutput.replace(/```json\n?/g, '').replace(/```\n?/g, '').trim();

let questions;
try {
  questions = JSON.parse(cleaned);
} catch (e) {
  // Fallback: extract the first [...] block
  const match = cleaned.match(/\[[\s\S]*\]/);
  if (match) {
    questions = JSON.parse(match[0]);
  } else {
    throw new Error(`Could not parse questions from LLM output: ${rawOutput}`);
  }
}

if (!Array.isArray(questions) || questions.length === 0) {
  throw new Error(`Expected array of questions, got: ${JSON.stringify(questions)}`);
}

return questions.map(q => ({ json: { question: String(q) } }));
```

- [ ] **Step 5: Add Code node to parse agent output**

Add a **Code** node after the AI Agent. Mode: **"Run Once for Each Item"** (default).

```javascript
const agentOutput = $input.item.json.output || '';
const question = $input.item.json.question || '';

// Strip markdown fences if the model added them
const cleaned = agentOutput.replace(/```json\n?/g, '').replace(/```\n?/g, '').trim();

let parsed;
try {
  // Find first {...} block in case the model added preamble
  const match = cleaned.match(/\{[\s\S]*\}/);
  parsed = JSON.parse(match ? match[0] : cleaned);
} catch (e) {
  // Parsing failed — return low-confidence fallback so the workflow doesn't break
  parsed = {
    answer: agentOutput,
    confidence: 'low',
    needs_review: true,
    reason: 'Agent output could not be parsed as structured JSON — manual review required.',
    sources: []
  };
}

return [{ json: { question, ...parsed } }];
```
